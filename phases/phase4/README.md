# Phase 4 — 내보내기 포맷 확장 (Export Formats)

> **목표**: HTML 외 OpenAPI, Postman Collection 등 표준 포맷으로 API 문서 내보내기.  
> **선행 조건**: Phase 1 완료 (config.output 확장자 기반 포맷 선택)

## 현재 문제점

- 출력이 HTML 단일 포맷으로 고정
- 스캔 결과 데이터(서버/클라이언트 엔트리)가 HTML 생성기에 직접 결합

## Feature 4-1: 출력 포맷 추상화

### 대상 파일
- 신규 `scripts/lib/exporters/types.ts`
- 신규 `scripts/lib/exporters/html-exporter.ts` (기존 html-generator.ts 래핑)
- `scripts/generate-api-docs.ts` (수정)

### 작업 내용

```typescript
// scripts/lib/exporters/types.ts
import { ApiEntry } from '../html-generator';
import { ClientApiEntry } from '../client-api-scanner';

export interface DocMeta {
  projectName: string;
  generatedAt: string;
  version?: string;
}

export interface DocExporter {
  format: string;
  extension: string;
  generate(
    serverEntries: ApiEntry[],
    clientEntries: ClientApiEntry[],
    meta: DocMeta
  ): string;
}
```

### 포맷 선택 로직

```typescript
// scripts/lib/exporters/registry.ts
import { DocExporter } from './types';
import { HtmlExporter } from './html-exporter';
import { OpenApiExporter } from './openapi-exporter';
import { PostmanExporter } from './postman-exporter';
import { JsonExporter } from './json-exporter';

const exporters: DocExporter[] = [
  new HtmlExporter(),
  new OpenApiExporter(),
  new PostmanExporter(),
  new JsonExporter(),
];

export function resolveExporter(outputPath: string): DocExporter {
  const ext = path.extname(outputPath).toLowerCase();
  const match = exporters.find(e => `.${e.extension}` === ext);
  return match ?? exporters[0]; // fallback: HTML
}
```

### CLI 확장

```bash
# 확장자 기반 자동 선택
fe-api-tracer --output docs/api.html       # → HTML
fe-api-tracer --output docs/openapi.yaml   # → OpenAPI
fe-api-tracer --output docs/api.json       # → Postman Collection

# 또는 명시적 지정
fe-api-tracer --format openapi --output docs/api.yaml
```

## Feature 4-2: OpenAPI (Swagger) 내보내기

### 대상 파일
- 신규 `scripts/lib/exporters/openapi-exporter.ts`

### 작업 내용

서버 라우트 + 클라이언트 호출 데이터를 OpenAPI 3.0 스펙으로 변환.

```typescript
export class OpenApiExporter implements DocExporter {
  format = 'openapi';
  extension = 'yaml';  // yaml 또는 json

  generate(serverEntries, clientEntries, meta): string {
    const spec = {
      openapi: '3.0.3',
      info: {
        title: meta.projectName,
        version: meta.version ?? '1.0.0',
        description: `Auto-generated by fe-api-tracer at ${meta.generatedAt}`,
      },
      paths: {},
    };

    // 서버 엔트리 → paths
    for (const entry of serverEntries) {
      const pathKey = this.toOpenApiPath(entry.route.routePath);
      // /api/user/:id → /api/user/{id}
      spec.paths[pathKey] = this.buildPathItem(entry);
    }

    // 클라이언트 엔트리 → paths (서버에 없는 것만 추가)
    for (const entry of clientEntries) {
      const pathKey = this.toOpenApiPath(entry.url);
      if (!spec.paths[pathKey]) {
        spec.paths[pathKey] = {};
      }
      this.mergeClientEntry(spec.paths[pathKey], entry);
    }

    return yaml.dump(spec);  // js-yaml 사용
  }
}
```

### 매핑 규칙

| fe-api-tracer | OpenAPI 3.0 |
|---------------|-------------|
| `routePath: '/api/user/:id'` | `paths['/api/user/{id}']` |
| `methods: ['GET', 'POST']` | `get: {...}`, `post: {...}` |
| `params[].required === true` | `required: true` |
| `params[].type === 'string'` | `schema: { type: 'string' }` |
| `params[].description` | `description` 필드 |
| `returns.type` | `responses.200.content.application/json.schema` |

### 의존 라이브러리
- `js-yaml` (YAML 출력용, 선택사항 — JSON으로도 출력 가능)

## Feature 4-3: Postman Collection 내보내기

### 대상 파일
- 신규 `scripts/lib/exporters/postman-exporter.ts`

### 작업 내용

Postman Collection v2.1 포맷으로 변환.

```typescript
export class PostmanExporter implements DocExporter {
  format = 'postman';
  extension = 'json';

  generate(serverEntries, clientEntries, meta): string {
    const collection = {
      info: {
        name: meta.projectName,
        description: `Auto-generated by fe-api-tracer`,
        schema: 'https://schema.getpostman.com/json/collection/v2.1.0/collection.json',
      },
      item: [],
      variable: [
        { key: 'baseUrl', value: 'http://localhost:3000', type: 'string' }
      ],
    };

    // 그룹별 폴더 생성
    const groups = this.groupEntries(serverEntries, clientEntries);
    for (const [groupName, entries] of groups) {
      const folder = {
        name: groupName,
        item: entries.map(e => this.toRequestItem(e)),
      };
      collection.item.push(folder);
    }

    return JSON.stringify(collection, null, 2);
  }

  private toRequestItem(entry): PostmanItem {
    return {
      name: `${entry.method} ${entry.url}`,
      request: {
        method: entry.method,
        url: {
          raw: '{{baseUrl}}' + entry.url,
          host: ['{{baseUrl}}'],
          path: entry.url.split('/').filter(Boolean),
        },
        body: entry.method !== 'GET' ? {
          mode: 'raw',
          raw: JSON.stringify(
            Object.fromEntries(entry.params.map(p => [p.name, `<${p.type}>`])),
            null, 2
          ),
          options: { raw: { language: 'json' } },
        } : undefined,
        description: entry.description,
      },
    };
  }
}
```

### 검증 기준
- 생성된 JSON을 Postman에서 Import하여 정상 로드 확인
- `{{baseUrl}}` 변수가 환경 변수로 분리됨
- 파라미터가 body에 템플릿으로 포함됨

## Feature 4-4: 중간 JSON 데이터 내보내기

### 대상 파일
- 신규 `scripts/lib/exporters/json-exporter.ts`

### 작업 내용

스캔 결과를 있는 그대로 JSON으로 저장. CI diff 모드(Phase 6)의 기반이 됨.

```typescript
export class JsonExporter implements DocExporter {
  format = 'json';
  extension = 'json';

  generate(serverEntries, clientEntries, meta): string {
    return JSON.stringify({
      meta,
      server: serverEntries.map(e => ({
        route: e.route.routePath,
        methods: e.endpoint.methods,
        description: e.endpoint.description,
        params: e.endpoint.params,
        returns: e.endpoint.returns,
        source: e.route.relativePath,
      })),
      client: clientEntries.map(e => ({
        method: e.method,
        url: e.url,
        description: e.description,
        params: e.params,
        returns: e.returns,
        source: e.sourceFile,
        functionName: e.functionName,
        group: e.group,
        usage: e.usageChain,
      })),
    }, null, 2);
  }
}
```
